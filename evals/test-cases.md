# Evals: Test Cases

Use these test cases to see if templates produce consistent, high-quality outputs.

## Test Case 1 — PRD (simple)
Input:
- Feature idea: “Add ‘Save for later’ to checkout”
- User: frequent shoppers
- Baseline: checkout conversion 42%

Expected:
- Clear success metric + guardrails
- MVP scope limited
- Risks include cannibalization / confusion

## Test Case 2 — KPI diagnosis

- KPI: D1 retention dropped 8% WoW

Expected:
- Facts vs hypotheses split
- First cuts: platform, version, cohort, crash rate
- Mitigations suggested
