# AI PM Prompt Playbook

A practical prompt engineering playbook for **AI Product Managers**.

This repo is designed to be:
- **Copy-pasteable** (templates you can use today)
- **Explainable** (why each technique works)
- **Eval-driven** (so prompts improve, not just change)

## What’s inside

- **Framework** → the default mental model for writing prompts
- **Techniques** → common patterns (with PM examples)
- **Templates** → ready-to-use prompts for core PM outputs
- **Playbooks** → end-to-end workflows (idea → PRD, funnel drop analysis, etc.)
- **Evals** → rubric + test cases to score prompt quality

## Start here

1) Read the framework:
- `framework/6-layer-prompt-stack.md`

2) Use templates:
- `templates/prd-generator.md`
- `templates/kpi-diagnosis.md`

3) When outputs drift, use evals:
- `evals/scoring-rubric.md`
- `evals/test-cases.md`

## Guiding principle
Prompt engineering is product strategy in disguise.

If you want to contribute:
- Keep prompts **structured**
- Be explicit about **inputs/constraints/output format**
- Add or update **test cases** when you change a core template
